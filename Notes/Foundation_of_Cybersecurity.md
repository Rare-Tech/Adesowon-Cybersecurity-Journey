Foundation of Cybersecurity – Summary
1. Security Domains
   
Security & Risk Management → governance, policies, and risk assessment.
Asset Security → protecting data and information assets.
Security Architecture & Engineering → designing secure systems.
Communication & Network Security → safeguarding data in transit.
Identity & Access Management (IAM) → controlling access rights.
Security Assessment & Testing → audits, scans, and evaluations.
Security Operations → monitoring, incident response, recovery.
Software Development Security → secure coding and application protection.

2. Benefits of Cybersecurity

Protects sensitive data (PII/SPII).
Maintains trust and reputation.
Ensures compliance with regulations.
Reduces financial and operational risks.

3. Primary Analyst Duties
   
Monitoring → logs, alerts, and traffic.
Responding → assisting with incidents.
Documenting → recording findings for accountability.

4. Key Cybersecurity Terms & Concepts
   
CIA Triad → Confidentiality, Integrity, Availability.
Threats & Vulnerabilities → dangers and weaknesses.
Defense in Depth → layered security.
Least Privilege → minimal access rights.

5. Transferable & Technical Skills
   
Transferable: problem‑solving, communication, teamwork, attention to detail.
Technical: log analysis, SIEM tools, vulnerability scanning, scripting basics.

6. PII vs SPII
   
PII (Personally Identifiable Information): data that identifies individuals (name, email).
SPII (Sensitive PII): highly sensitive data (SSN, medical records, financial info).

7. Evolution of Security
   
From basic perimeter defenses → to network firewalls → to modern layered defenses with cloud, AI, and proactive monitoring.

9. Attack Types & Roles
    
Phishing, Malware, Ransomware, Social Engineering, DDoS.
Each plays a role in exploiting vulnerabilities and testing defenses.

9. Security Frameworks
    
Purpose: provide structured guidelines for managing security.
Components: policies, controls, processes (e.g., NIST, ISO 27001).

10. Tools & Programming Languages
    
Tools: SIEM (Splunk, QRadar), IDS/IPS, firewalls, vulnerability scanners.
Languages: Python (automation), SQL (querying), Bash (scripting).

11. Ethics in Cybersecurity
    
Principles: integrity, confidentiality, accountability, respect for privacy.
Analysts must act responsibly, avoid misuse of data, and uphold trust.
